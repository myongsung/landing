FIELD PROOF

탁상에서 끝나지 않도록. 현장에서 바로 쓰는 규정준수 런타임. COA는 생성되고, 위반은 반례로 드러나며, 절차는 수리되어 끝까지 완수됩니다.

01 / ON-DEVICE
외부 LLM/API 없이도 작동하도록 설계된 오프라인 런타임. 단말에서 정책 번들을 로드하고, COA를 생성·검증합니다.
02 / COUNTEREXAMPLE
위반은 숨기지 않습니다. 반례로 위치/규칙/사유를 명확히 제시하고, 수리로 이어집니다.
03 / AUDIT PACK
“왜 그렇게 했는가”를 남깁니다. 근거 매핑(span→rule_id) + COA + 반례 + 수리 이력으로 감사 패키지를 구성합니다.

MISSION

국방 리스크의 본질은 절차 누락과 근거 부재입니다.
REDCLX는 규정을 논리 제약식으로 고정하고, 실행 로그로 준수 사실을 증명합니다.

AUDIT ENGINE

REDCLX는 “추천”이 아니라 규정준수 여부를 판단하고 증명하는 실행 엔진입니다.
정책 번들(Policy Bundle)을 기반으로 COA 그래프를 생성·검증하고, 위반 시 반례와 수리 이력을 남겨 설명 가능한 절차를 제공합니다. 외부 LLM/API 없이도 동작하도록 설계되어, 오프라인 작전/훈련 환경에서 적용 가능합니다.

Integrity Proof
근거-룰-조치 간 일치성 검증
Counterexample
위반 위치/사유를 반례로 산출
Offline Runtime
네트워크 없이 단말에서 실행
01 / LOGIC COMPILATION
자연어 규정을 룰과 온톨로지로 정규화하고, 단말 실행을 위해 Policy Bundle(JSON)로 컴파일합니다.
02 / GENERATION & CHECK
케이스 입력(상황/제약/자원)에 따라 COA 그래프를 생성하고, 각 단계가 정책 룰을 만족하는지 검증합니다.
03 / COUNTEREXAMPLE REPAIR
위반이 발견되면 위반 룰과 위치를 반례로 산출합니다. 반례를 입력으로 수리 연산을 적용해 COA를 준수 가능한 그래프로 보정합니다.
OUTPUT
Verified COA · Action Graph · Audit Pack
Verified Action Graph
  • Flow: 규정과 정합성이 검증된 행동 절차
  • Logic Link: 단계별 근거(문장/span) 자동 연결
  • Risk Guard: 위반 시도 탐지 및 보정
Audit Evidence Pack
  • Proof: 근거·룰·행동 연결로 책임소명
  • Traceability: 상황→조치→종결까지 추적
  • Repair Trace: 반례·수리 이력 포함
Technical Core

생성과 검증을 분리하여, 오프라인 환경에서도 신뢰 가능한 결과를 남기는 것을 목표로 합니다.

  • Policy Compilation: 규정 문장 → 온톨로지/룰 → 실행용 번들(JSON)
  • Constraint Verification: MUST/MUST_NOT/MUST_BEFORE/IF_THEN 룰 기반 검증
  • Counterexample-Guided Repair: 위반 반례 기반 그래프 자동 수리
  • Offline Runtime: 네트워크·외부 API 없이 단말에서 결정적 실행

PROCUREMENT FIT

방위사업청/국방부 실증·도입 관점에서 중요한 것은 “멋진 데모”가 아니라 운영 가능성, 보안, 산출물입니다. REDCLX는 오프라인 런타임 + 정책 번들 + 감사 패키지를 기본 단위로 제공합니다.

OFFLINE-READY
네트워크 불가/제한 환경을 전제로 설계. 외부 API 의존 없이 정책 번들로 동작.
AUDIT DELIVERABLES
COA, 반례, 수리 이력, 근거 매핑으로 검토/감사 가능한 산출물 제공.
SCOPED PoC
SOP 단위로 범위를 고정하고 준수율/누락률/검증 통과율로 평가.
POLICY BUNDLE
규정 업데이트를 “코드 배포”가 아니라 번들 교체로 단순화.
INTEGRATION
로그/리포트는 JSON 기반으로 상위 시스템 연동에 유리.
OPERATIONAL CONTROL
“왜 이 단계인가”를 남겨 현장 책임소명과 교육에 활용.
STEP 01 대상 SOP/규정 선정 (범위 고정)
STEP 02 온톨로지/룰 제작 → 번들 컴파일
STEP 03 오프라인 단말 배포 → 준수/검증 지표 측정
STEP 04 감사 패키지 산출 → 적용 확대
오프라인 환경에서 업데이트는 어떻게 하나요?
정책은 Policy Bundle(JSON)로 배포됩니다. 규정 개정 시 번들 교체로 적용을 단순화합니다.
감사 대응 산출물은 어떤 형태로 제공되나요?
COA(Action Graph) + 위반 반례 + 수리 이력 + 근거 매핑(span→rule_id)로 구성된 Audit Evidence Pack을 제공합니다.
훈련관리자/조교 사용성을 어떻게 담보하나요?
현장 워크플로우를 기준으로 입력을 최소화하고, 단계 UI/체크리스트로 실행 부담을 낮춥니다. 모바일 최적화를 기본 설계로 둡니다.
